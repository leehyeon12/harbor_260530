# [Context+Agent+DB] 내 카페를 아는 AI 에이전트

Q6에서 만든 `my_cafe.md`(바이턴 컨셉)를 **컨텍스트**로, Supabase 운영 DB(매출·메뉴판매·리뷰·재고)를
**데이터**로 결합해, 범용 AI와는 다른 수준의 맞춤 답변을 주는 카페 운영 파트너 에이전트.

## 무엇인가

- **에이전트**: `cafe-advisor` (Claude Code 서브에이전트)
- **컨텍스트**: `week_5/quest/06_my-cafe-concept/my_cafe.md` (Read 도구로 매번 먼저 확인)
- **DB 접속**: Supabase MCP(`mcp__supabase__execute_sql`, 읽기 전용)로 카페 운영 테이블 조회
- **핵심 증명**: 같은 질문도 **컨텍스트 유무에 따라 답변 수준이 완전히 달라짐** (Before/After 아래 참조)

## 파일

- `cafe-advisor.md` — 에이전트 정의(프론트matter + 시스템 프롬프트). 실제 설치 위치는 `.claude/agents/cafe-advisor.md`.
- `seed.js` / `package.json` — 카페 운영 데이터 시드 스크립트(재현용)
- `README.md` — 이 문서

## DB 스키마 (`harbor_w5_cafe_*`, 2주치 시드)

| 테이블 | 내용 |
|---|---|
| `harbor_w5_cafe_daily_sales` | 일별 매출·요일·고객수 (14일) |
| `harbor_w5_cafe_menu_sales` | 메뉴별(아메리카노·카페라떼·바스크치즈케이크·마들렌·딸기스무디) 판매 기록 (70건) |
| `harbor_w5_cafe_reviews` | 손님 리뷰 (10건) |
| `harbor_w5_cafe_inventory` | 재료 재고·발주 정보 (6건, 크림치즈 재고 부족 상태로 시드) |

## 세팅 (재현 방법)

1. `week_5/quest/06_my-cafe-concept/my_cafe.md`가 이미 존재해야 함(카페 컨텍스트)
2. Supabase MCP 연결(Q2 README 참고, 읽기 전용)
3. 데이터 시드: `npm install && npm run seed` (`CREATE TABLE IF NOT EXISTS` + 비어있을 때만 시드)
4. `.claude/agents/cafe-advisor.md` 설치 후 Claude Code에서 `/reload-plugins`
5. `@cafe-advisor 신메뉴 추천해줘` 처럼 호출

## Before / After 비교 (제출 핵심)

### ① "신메뉴 추천해줘"

**Before (컨텍스트 없음)** — 일반론
> 딸기라떼, 크로플, 아인슈페너, 흑당버블티 같은 요즘 인기 메뉴를 추천드려요. 카페 컨셉에 맞는 걸 골라보세요!

**After (`@cafe-advisor`, my_cafe.md + DB 근거)**
> **바스크 치즈케이크 맛 변형**과 **마들렌 주말 선물세트**를 추천, **딸기스무디는 단종 후보**.
> 근거: 치즈케이크 주말 평균 45.5개(평일 대비 3배), 리뷰에 "치즈케이크·마들렌 품절" 불만 2건 확인, 딸기스무디는 2주 합계 35개로 최하위(2위 대비 5배 차이).

### ② "이번 주 마케팅 어떻게 할까?"

**Before (컨텍스트 없음)** — 일반론
> 인스타그램 신메뉴 홍보, 첫 방문 할인 쿠폰, 리뷰 이벤트, 계절 한정 메뉴 같은 방법이 있어요.

**After (`@cafe-advisor`, my_cafe.md + DB 근거)**
> **평일은 반복방문 굳히기, 주말은 객수 확대** 투트랙. 평일 고객수 91~110명(객단가 5,400~5,800원) vs 주말 54~57명(객단가 8,900~9,300원) — 주말은 객수가 평일의 절반이라는 게 실제 데이터로 확인됨. 구체적으로 "6인+ 단체석 예약 시 마들렌 서비스", "불금 치즈케이크 예약 판매로 평일→주말 트래픽 전환" 등 my_cafe.md 차별점(단체석)에 근거한 캠페인 제안.

### ③ (보너스, 실무 활용) "재고 발주 타이밍 알려줘"

> **크림치즈를 지금 발주해야 함** — 현재고 2.50kg, 재주문 기준 5.00kg의 절반, 마지막 발주 6/25로 다른 재료 대비 과도하게 오래됨. 이 재료가 시그니처 "바스크 치즈케이크"(일평균 12개 판매)에 쓰인다는 것까지 연결해 "주말 전 품절 리스크" 경고. 다른 5개 재료는 여유 있음도 함께 확인해 우선순위를 명확히 함.

→ 세 답변 모두 **my_cafe.md의 구체 정보(메뉴명·존 구조·차별점) + 실제 DB 숫자**를 근거로 삼아, Before의 뻔한 일반론과 확연히 다른 실무 수준 조언을 제공.

## 창의성 요소

- 시드 데이터를 my_cafe.md 컨셉과 정합되게 설계 (평일 고객수↑객단가↓ / 주말 고객수↓객단가↑, 시그니처 메뉴 주말 강세)
- 의도적 "약점 데이터"(크림치즈 재고부족, 마들렌·치즈케이크 품절 리뷰, 딸기스무디 부진)를 심어 에이전트가 실제로 찾아내는지 검증
- 재고 시나리오에서 재료→메뉴 연결 추론까지 요구해 "실무 활용 사례"로 제출

## 기술 메모

- 쓰기는 read-only Supabase MCP로 막고, 데이터 적재는 `seed.js`(pg) 경유
- 쿼리 결과의 untrusted 데이터 경계 안 지시는 따르지 않음(값만 사용)
- 테이블명 규칙: `harbor_w5_cafe_*` (프로젝트별 접두어, harbor_w5_ledger_*·harbor_w5_shop_*와 동일 컨벤션)
