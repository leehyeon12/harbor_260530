# 관심분야 자동 리서치 스킬

주제 하나를 받아 웹에서 자동으로 조사하고, 매번 똑같은 포맷의 마크다운으로 정리해 주는 스킬.
"에이전트는 한 번, 스킬은 매번" — 내가 좋아하는 정리 방식을 박제해 재현한다.

## 기본 저장 폴더

- 산출물: `week_5/quest/05_research-skill/research-<주제슬러그>-<YYYY-MM-DD>.md`
- 다른 프로젝트에서 쓰면 그 프로젝트의 적절한 위치로 바꾼다. (개인/비공개 리서치면 gitignore 폴더에 저장)

## 필요한 도구

- **Playwright MCP**(브라우저): 사이트 탐색 — 필수
- **Notion MCP**: 결과 페이지 업로드 — 있으면 사용(보너스), 없으면 건너뛴다

## 작업 순서

### 1) 주제·키워드 확인
- 사용자에게 **리서치 주제/키워드**를 묻는다. (예: "카페 창업 트렌드", "SAP 채용공고", "○○동 상권")
- 이미 주제를 줬으면 바로 진행. 필요하면 검색 각도 1~2개를 스스로 잡는다(예: 트렌드/비용/메뉴).

### 2) 브라우저로 사이트 3곳+ 탐색 ⭐(이 스킬의 핵심)
- Playwright MCP로 **서로 다른 각도의 검색/페이지 3곳 이상**을 연다.
  (예: 네이버/구글 검색결과, 관련 블로그·매거진·뉴스, 업계 포털)
- 각 페이지에서 `browser_evaluate`로 본문 텍스트를 뽑아 핵심 정보를 수집한다.
- **각 사이트마다 스크린샷 1장씩(총 3장+)** 남긴다 (`browser_take_screenshot`).
- 로딩이 느리면 대기 후 재시도한다.

### 3) research-<주제>-<날짜>.md 생성 (아래 포맷 고정)
```
# <주제> 리서치

**수집일**: <YYYY-MM-DD>
**주제/키워드**: <주제>
**탐색 사이트(3곳)**: <출처 목록>

## 출처별 핵심
### ① <출처1>
- (핵심 3줄)
### ② <출처2>
- (핵심 3줄)
### ③ <출처3>
- (핵심 3줄)

## 한 줄 요약
<전체를 한 문장으로>

## 다음 액션
- [ ] <실행 항목 2~3개>
```
- 사이트마다 **정확히 3줄**의 핵심을 담는다(과하게 길게 쓰지 않는다).

### 4) 노션 공유 (선택 · 보너스)
- Notion MCP가 있으면 `notion-create-pages`로 위 내용을 페이지로 올린다.
  - 제목 = "<주제> 리서치 (<날짜>)", 아이콘 이모지 하나, 본문 = md 내용(맨 위 제목 제외).
  - 상위 페이지를 특정 못 하면 parent 생략(워크스페이스 개인 페이지로 생성).
- 생성된 페이지 URL을 사용자에게 알려준다.

### 5) 결과 보고
- 저장한 md 경로, 노션 URL(있으면), 스크린샷 개수를 표로 요약 보고한다.

## 규칙 / 예외

- **사이트는 반드시 3곳 이상.** 한 곳만 보고 정리하지 않는다.
- 특정 사이트가 봇 차단/로그인 요구로 막히면 다른 접근 가능한 출처로 대체한다.
- 포맷 필드(제목·수집일·출처별 3줄·한줄요약·다음액션)를 **매번 동일하게** 유지한다 — 재현성이 이 스킬의 존재 이유.
- 파일명은 소문자+하이픈 슬러그(`research-cafe-trend-2026-07-10.md`).
