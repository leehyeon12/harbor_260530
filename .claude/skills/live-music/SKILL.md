# Live Music 다운로더

유튜브의 라이브 공연·음악 영상을 받아 **영상(mp4) + 음원(mp3)** 을 저장하고,
파일명을 사람이 보기 좋은 형태로 정리한다.

## 기본 저장 폴더

사용자가 위치를 지정하지 않으면 아래에 저장한다:

```
week_5/practice/03_my-live-music/
```

(개인 음원 수집 폴더. `.gitignore`에 등록되어 git 추적에서 제외됨.)
사용자가 다른 폴더를 말하면 그쪽에 저장한다.

## 작업 순서

### 1) 의존성 확인 (필요할 때만 설치)

`yt-dlp` 와 `ffmpeg` 가 필요하다. 스크립트가 없으면 알아서 알려주지만,
미리 확인하려면:

```bash
export PATH="/opt/homebrew/bin:$PATH"
which yt-dlp ffmpeg
```

없으면 `brew install yt-dlp ffmpeg` 로 설치한다(맥 기준). 설치는 처음 한 번이면 충분하다.

### 2) 제목을 읽고 깔끔한 파일명을 판단 (이 스킬의 핵심)

유튜브 원본 제목은 대개 지저분하다 — "역대급 최고의 공연!", "[한/영 자막]",
채널명, 이모지 등이 붙는다. `artist`/`track` 메타데이터 필드는 공식 채널이 아니면
대부분 `NA`라 **믿지 말고**, 제목 텍스트를 직접 읽어 판단한다.

먼저 제목을 가져온다:

```bash
export PATH="/opt/homebrew/bin:$PATH"
yt-dlp --skip-download \
  --print "title=%(title)s" \
  --print "uploader=%(uploader)s" \
  "<URL>"
```

그다음 **아래 표준 형식**으로 이름을 만든다:

```
아티스트 - 곡명 (공연/앨범 또는 연도)
```

**예시:**

Input(원본 제목): `역대급 최고의 공연! [한/영 자막] Queen(퀸) 의 'Radio GaGa' Live aid concert`
Output(파일명): `Queen - Radio GaGa (Live Aid 1985)`

Input: `BTS (방탄소년단) 'Dynamite' Official MV`
Output: `BTS - Dynamite (Official MV)`

Input: `아이유(IU) - 밤편지 [유희열의 스케치북]`
Output: `IU - 밤편지 (유희열의 스케치북)`

판단 원칙:
- 홍보성 수식어("역대급", "최고의", "레전드"), 자막 표기("[한/영 자막]"),
  이모지는 버린다.
- 아티스트명은 널리 쓰이는 표기로 통일한다(예: `Queen(퀸)` → `Queen`,
  `아이유(IU)` → `IU`). 한국 아티스트는 상황에 맞게 한글/영문 중 자연스러운 쪽을 쓴다.
- 괄호 안에는 공연명·앨범명·연도 등 곡을 특정하는 정보를 넣는다.
  라이브 공연이면 유명한 연도를 알면 함께 적어준다(예: Live Aid는 1985).
- 애매하면 임의로 확정하지 말고 후보 이름을 사용자에게 한 번 확인한다.

### 3) 다운로드 + 음원 추출

헬퍼 스크립트가 다운로드·추출·파일명 저장을 모두 처리한다.
2)에서 만든 이름을 그대로 넘긴다:

```bash
bash "<이 스킬 경로>/scripts/download.sh" \
  "<URL>" \
  "<저장 폴더>" \
  "<깔끔한 파일명(확장자 없이)>" \
  both
```

- 마지막 인자 `both` = 영상+음원 모두. `audio` = mp3만, `video` = mp4만.
- 사용자가 "음원만", "mp3만" 이라고 하면 `audio` 를 쓴다.
- 스크립트가 파일명의 슬래시(`/`) 등 위험 문자를 안전하게 처리하고,
  끝나면 저장된 파일 목록을 출력한다.

### 4) 결과 보고

저장 위치와 파일명(mp4/mp3), 용량을 표로 간단히 알려준다.
저작권이 있는 콘텐츠이므로 **개인 소장·감상 용도**임을 처음 한 번 정도 가볍게 짚어준다.

## 파일명만 정리해달라는 경우

이미 받아둔 파일의 이름 정리만 요청하면, 2)의 명명 규칙(`아티스트 - 곡명 (공연/앨범)`)을
적용해 `mv` 로 바꾼다. 같은 폴더에 여러 곡이 쌓일 예정이면 규칙을 일관되게 유지한다.

## 여러 개를 한 번에

여러 URL을 주면 각각에 대해 2)~3)을 반복한다. 재생목록(playlist) URL이면
`yt-dlp` 가 자동으로 여러 개를 받으므로, 개별 파일명 정리가 필요하면
받은 뒤 제목을 확인해 이름을 맞춰준다.
